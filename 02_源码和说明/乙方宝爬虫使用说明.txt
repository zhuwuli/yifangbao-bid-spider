# 乙方宝招标信息爬取与整理说明书

当前说明对应版本：1.2.0（2026-07-17）。

## 适用场景

本工具用于从乙方宝/千里马企业站搜索招标信息，并把结果追加到 Excel 表格中。当前配置面向：

- 地区：山东济南、莱芜
- 关键词：监测、水土保持、测绘、测量、绿色建筑评价、绿色建筑验收
- 输出文件：`2026-乙方宝招标信息统计.xlsx`，并自动生成 `2026-乙方宝招标信息统计-筛选后.xlsx`
- 输出结构：主表放摘要，`公告详情` sheet 放资质和公告长文本
- 主表项目名称链接乙方宝详情页，可见项目编号用于人工删除、恢复和跨表关联
- 默认只下载本次新增且通过最终筛选的公告 PDF，保存到 `公告/YYYYMMDD`

## 文件组成

建议打包时至少包含这些文件：

- `yfb_bid_spider.py`：检索、详情解析、终筛和 Excel 写入
- `yfb_spider_app.py`：桌面软件界面与进度展示
- `yfb_browser_auth.py`：浏览器检测、专用登录和登录状态读取
- `2026-乙方宝招标信息统计.xlsx`：不含真实业务数据的 Excel 模板
- `requirements.txt`：固定版本的运行依赖
- `requirements-dev.txt`：固定版本的打包依赖
- `乙方宝爬虫使用说明.md` / `.txt`：本说明书

可选文件：

- `2026-乙方宝招标信息统计-审核清理版.xlsx`：人工审核后的清理版结果
- `审核清理删除项目清单.txt`：清理版删除记录

## 环境要求

### 直接使用 EXE

- Windows 10/11
- 能访问乙方宝网页
- 已登录乙方宝账号
- 不需要安装 Python
- 不需要创建或激活 `.venv`

### 从源码运行或重新打包

- Python 3.10 或更高版本，推荐 Python 3.12
- 当前发布构建验证环境：Python 3.12.7
- 运行依赖：openpyxl 3.1.5、websocket-client 1.8.0、requests 2.32.3
- 打包依赖：PyInstaller 6.21.0

项目使用上一级目录中的 `.venv` 隔离依赖。首次配置时，在项目根目录运行：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r ".\02_源码和说明\requirements.txt"
```

无需激活虚拟环境，后续命令直接调用 `.venv\Scripts\python.exe` 即可。

## 浏览器自动登录

1. 打开软件，在“登录浏览器”中选择自动选择或指定浏览器。
2. 首次使用点击“登录乙方宝”。
3. 在弹出的专用浏览器窗口中正常登录乙方宝。
4. 软件检测到登录状态后会自动关闭该窗口并保存状态。
5. 以后直接点击“开始爬取”，软件会自动读取保存的 Cookie。
6. 登录失效后再次点击“登录乙方宝”；软件会验证接口，旧 Cookie 不会再被误判为登录成功。

软件自动检测 Chrome、Brave、Edge，并按此顺序优先选择，也可以手动选择 Chromium 内核浏览器 EXE。每种浏览器使用 `browser_data` 下的专用资料目录，不读取日常浏览器配置。

手工 Cookie 是备用方式：在乙方宝搜索页按 `F12`，进入“网络”，执行一次搜索，打开 `queryZBInfo` 请求，在“请求标头”中复制完整 Cookie，粘贴到软件的“Cookie（备用）”输入框。

注意：Cookie 是登录凭证，不要发给无关人员。网站使登录状态失效后仍需重新登录，但无需再手工查找 Cookie。

## 基本运行命令

在项目根目录打开 PowerShell：

```powershell
$env:YFB_COOKIE='这里粘贴你的Cookie'
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_bid_spider.py" --days 31
```

指定其他 Excel：

```powershell
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_bid_spider.py" --days 31 --xlsx "D:\项目\2026-乙方宝招标信息统计.xlsx"
```

演练模式，不写入 Excel：

```powershell
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_bid_spider.py" --days 31 --dry-run
```

## 桌面软件用法

如果不想敲命令，可以直接运行桌面界面：

```powershell
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_spider_app.py"
```

或者双击：

```text
启动乙方宝爬虫工具.bat
```

界面里需要填写：

- `登录浏览器`：自动选择，或指定检测到的 Chrome、Brave、Edge
- `登录乙方宝`：首次登录或登录失效后重新登录
- `Cookie（备用）`：自动登录不可用时手工粘贴 Cookie
- Excel 主表按表头和 `Sheet1` 名称识别，不受保存时当前活动工作表影响；空表从第 2 行写入
- `Excel 文件`：选择要写入的表格
- `时间范围`：近 7 天、近 1 个月、近 3 个月或自定义天数
- `演练模式`：勾选后只测试，不写入 Excel

点击 `开始爬取` 后，界面会持续显示滚动进度条、当前阶段和详细日志。列表检索、详情获取、串行补抓、Excel 写入和备份均有反馈；耗时较长时不要重复点击按钮或强制关闭软件。

## 打包成 EXE

在 Windows 电脑上双击：

```text
打包成EXE.bat
```

脚本会自动使用项目根目录的 `.venv`，安装 `requirements-dev.txt` 中固定版本的依赖，然后运行 PyInstaller。手动打包：

```powershell
cd "项目根目录"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r ".\02_源码和说明\requirements-dev.txt"
cd ".\02_源码和说明"
..\.venv\Scripts\python.exe -m PyInstaller --noconfirm --clean --windowed --name "乙方宝招标信息爬取工具" --hidden-import yfb_bid_spider yfb_spider_app.py
```

打包完成后，程序在：

```text
dist\乙方宝招标信息爬取工具
```

把整个文件夹压缩为 ZIP 后发布。使用者解压并运行 EXE 即可，不需要 Python 或 `.venv`。

- 最近 1 个月：`--days 31`
- 从 4 月 14 日之后到 6 月 26 日：`--days 73`

- 短范围用乙方宝发布时间“全部”，再本地过滤。

## 当前搜索逻辑

关键词依次搜索：

- 监测
- 水土保持
- 测绘
- 测量
- 绿色建筑评价
- 绿色建筑验收

地区限制：

- 济南
- 莱芜

检索组合共 4 种：

- 全文检索 + 智能检索
- 全文检索 + 精准检索
- 标题检索 + 智能检索
- 标题检索 + 精准检索

对应脚本参数：

- `filterCondition=1`：全文检索
- `filterCondition=2`：标题检索
- `searchType=1`：智能检索
- `searchType=2`：精准检索

## 加速与完整性保护

- 6 个关键词分别执行 4 种检索组合，共 24 个列表检索组合。
- 列表组合固定 3 路并发；每个组合内部仍按页顺序完整翻页。
- 每个网络线程复用连接，单次请求最多重试 3 次。
- 并发失败的列表组合会串行整组补抓；补抓仍失败时终止本次任务，避免生成看似成功但实际缺项的结果。
- 新项目详情固定 5 路并发；失败项目随后串行补抓。
- 详情最终仍失败时保留列表信息，并在备注中标记人工核对，不会静默删除项目。
- 并发只改变请求速度，不改变检索组合、分页范围、终筛规则和 Excel 写入顺序。

## 结果判断逻辑

软件采用“原始表宽口径保留、筛选后表统一终筛”的两阶段逻辑：

- 列表和详情阶段只执行时间、地区、去重和检索来源等必要控制，不因标题或摘要看起来不相关而提前删除候选。
- 原始 Excel 尽可能保存搜索接口返回的候选，便于追溯漏报和规则边界。
- 筛选后 Excel 执行严格业务规则，并生成 `筛选删除记录` 工作表，逐条记录被删除项目及原因。

终筛规则：

- 标题命中水土保持、水保、绿色建筑评价、绿色建筑验收、绿建评价、绿建验收、绿色建筑、测绘、测量、多测合一、地形图、规划核实、房产实测、国土变更调查、不动产测绘、竣工测量、放线测量、验线测量、基坑监测、深基坑监测、第三方监测、专项监测、变形监测、沉降观测、沉降监测、结构监测、运营期结构监测或监测服务时直接保留。
- 标题未命中但正文命中水土保持或绿色建筑相关业务词时直接保留。
- 测绘/测量类正文需同时命中测绘测量业务词和“测绘乙级”等资质表达。
- 监测类正文需同时命中监测业务词和工程测量、工程勘察、岩土工程、监测业绩等资质或业绩表达。
- 标题和正文会先去除空白字符再匹配，避免“结构监测 服务”等写法漏检。
- 物探、地下物探、工程物探、探测目前未纳入正式保留规则，应通过原始表或 `筛选删除记录` 人工确认。

## Excel 输出结构

主表列结构：

1. 序号
2. 日期
3. 项目名称
4. 建设单位
5. 项目位置
6. 资质
7. 报名时间
8. 投标截止时间
9. 基本情况
10. 相关性
11. 备注
12. 项目编号，例如 `YFB-20260717-001`
13. 下载

主表为了方便查看，不直接放长文本：

- 项目名称：点击打开乙方宝项目详情页。
- 资质列显示 `公告资质`，按项目编号动态跳转到 `公告详情`。
- 备注列显示短提示，按项目编号动态跳转到 `公告详情`。
- 下载列显示 `下载PDF`，点击乙方宝公告导出地址。
- 项目编号保持可见，排序和删除其他行后不会重新编号。

`公告详情` sheet 包含：

- 主表序号
- 项目名称
- 内容类型
- 详细内容
- 返回主表
- 项目编号（可见，与主表一致）
- 链接键（隐藏，软件维护）

## 推荐使用流程

1. 关闭准备写入的 Excel/WPS 文件，启动软件并确认登录状态。
2. 选择 Excel、时间范围；首次运行可勾选演练模式检查日志，正式采集时取消勾选。
3. 等待列表检索、详情获取和 Excel 写入全部完成，不要在中途关闭软件。
4. 默认勾选“自动下载筛选后新增公告 PDF”；完成后可点击“打开公告文件夹”。
5. 优先打开自动生成的 `-筛选后.xlsx` 查看保留结果。
6. 检查 `筛选删除记录` 中的边界项目；对资质、单位、时间为空或备注提示人工核对的项目，点击项目名称打开原公告复核。
7. 原始表、筛选后表和自动备份均应保留，便于后续审核和规则回溯。

写入完成后，主表按日期升序排列并重新编号；相同日期继续合并显示，筛选后表保持相同顺序。

## 人工删除和恢复项目

项目编号采用 `YFB-YYYYMMDD-NNN`，例如 `YFB-20260717-001`。编号生成后保持不变，同一项目在 `Sheet1`、公告详情资质行和公告详情备注行中必须一致。

人工二次筛选时：

1. 在 `Sheet1` 删除不需要的项目行。
2. 在 `公告详情` 中搜索相同项目编号，删除对应的资质和备注行。
3. 其他项目的链接通过项目编号动态查找，不会因为行号上移而串到相邻项目。

恢复误删项目时，从自动备份复制主表行和对应详情行，并保留原项目编号。原编号确实找不到时，可使用一个尚未占用的新编号，例如 `YFB-20260717-099`；不要给两个不同项目使用同一个编号。

## 公告 PDF 归档

- 软件只下载本次新抓取、写入后又通过最终筛选的公告，不重复下载历史项目。
- 保存目录为 `Excel所在目录/公告/YYYYMMDD`，日期是运行软件的日期。
- 文件名为 `项目编号_项目名称.pdf`。
- 下载前检查乙方宝导出权限并验证文件以 `%PDF-` 开头；错误页面不会保存成 PDF。
- 导出受会员、地区和每日额度限制。失败时 Excel 仍保留下载链接，具体原因显示在 `[PDF]` 日志中。
- 不需要自动下载时，取消软件中的勾选；命令行可使用 `--no-download-pdf`。

## 常见问题

### 认证失败

现象：提示 `认证失败，无法访问系统资源`。

处理：点击软件中的“登录乙方宝”重新登录；如自动登录不可用，再使用备用 Cookie 输入。

### 公告 PDF 下载失败

先查看日志中的 `[PDF]` 提示。常见原因是乙方宝登录失效、账号没有 PC 导出权限、地区权限不符或当天导出额度不足。重新登录只能解决登录失效，不能绕过会员和额度限制。

### Excel 保存失败

现象：`PermissionError`。

处理：关闭正在打开的 Excel 文件后重新运行。

### 结果里仍然有无关项目

原因：为了尽可能全面，全文检索和疑似相关会保留一些边界项目。

处理：筛选 `相关性=疑似相关`，人工检查；必要时把新发现的噪声标题词加入脚本的 `NOISE_TITLE_TERMS`。

### 漏掉项目

可能原因：乙方宝接口没有返回、Cookie 权限变化、关键词没有覆盖、地区筛选不一致。

处理：先在网页手动搜索确认，再把标题或关键词补充到脚本逻辑。

## 维护位置

主要改这些位置：

- `SEARCH_KEYWORDS`：搜索关键词
- `AREA_IDS`：地区编号
- `TITLE_KEEP_TERMS`：终筛标题直接保留词
- `WATER_GREEN_TERMS`：水土保持/绿色建筑正文直接保留词
- `SURVEY_QUALIFICATION_PATTERN`：测绘测量资质匹配规则
- `MONITOR_QUALIFICATION_PATTERN`：监测类资质/业绩匹配规则
- `PROTECTED_TITLE_TERMS`、`NOISE_TITLE_TERMS`、`STRICT_NOISE_TITLE_TERMS`：相关性标记和噪声识别词表
