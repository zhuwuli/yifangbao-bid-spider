# 乙方宝招标信息爬取工具

一款面向 Windows 的乙方宝/千里马企业站招标信息采集与 Excel 整理工具。当前版本针对山东济南、莱芜地区，按“监测、水土保持、测绘、测量、绿色建筑评价、绿色建筑验收”等关键词检索，并提取项目名称、建设单位、资质要求、报名时间和投标截止时间等信息。

> 本项目为个人效率工具，与乙方宝、千里马及相关网站无官方关联。请遵守目标网站的服务条款、访问规则和适用法律，仅采集自己有权访问的信息。

## 主要功能

- 提供 Windows 桌面界面，也可通过 Python 命令行运行。
- 支持近 7 天、近 1 个月、近 3 个月及自定义时间范围。
- 同时执行“全文/标题检索 × 智能/精准检索”四种组合，合并并去重结果。
- 限定山东济南、莱芜地区，依次检索监测、水土保持、测绘、测量、绿色建筑评价、绿色建筑验收。
- 原始表采用宽口径写入，避免初筛误删；筛选后表再按业务规则严格保留。
- 自动提取建设单位、资格要求、报名时间和投标截止时间。
- 主表保留摘要，长文本写入“公告详情”工作表，并通过超链接互相跳转。筛选后文件额外生成“筛选删除记录”，记录被删项目和原因。
- 写入前按项目标题去重，保存前自动备份原 Excel。
- 每次写入后整个主表按日期升序排列并重新编号，相同日期继续合并显示；筛选后表保持相同顺序。
- 支持演练模式，只显示结果而不修改 Excel。
- 运行时显示滚动进度条，并在日志中展示列表页、关键词组合、详情解析和写入进度。
- 24 个列表检索组合固定使用 3 路并发，每个组合仍完整顺序翻页；失败组合会串行整组补抓，补抓仍失败则终止任务。
- 每个工作线程复用网络连接；公告详情固定使用 5 路并发，失败项目串行补抓，最终失败会保留列表信息并写入人工核对备注。

## 直接使用软件

1. 从 GitHub Releases 下载最新的 Windows 压缩包并完整解压。
2. 双击 `乙方宝招标信息爬取工具.exe`。
3. 在“登录浏览器”中使用自动选择，或选择软件检测到的 Chrome、Brave、Edge。
4. 首次使用点击“登录乙方宝”，在弹出的专用浏览器窗口中正常登录；软件检测成功后会自动保存浏览器登录状态。
5. 选择 Excel 和时间范围，点击“开始爬取”。以后软件会自动读取已保存的登录状态，不需要每次复制 Cookie。
6. 登录失效时，软件会提示重新登录；再次点击“登录乙方宝”即可。
7. 完成后打开 Excel，优先检查筛选后表和“筛选删除记录”。

软件使用独立的 `browser_data` 目录保存浏览器自身加密的登录资料，不读取日常浏览器配置。浏览器下拉框自动检测 Chrome、Brave、Edge，并按此顺序优先选择，也支持手动选择 Chromium 内核浏览器 EXE。

手工 Cookie 仍作为备用：在乙方宝搜索页按 `F12`，进入“网络”，执行一次搜索，打开 `queryZBInfo` 请求，在“请求标头”中复制完整 `Cookie` 并粘贴到软件的“Cookie（备用）”输入框。

Cookie 相当于临时登录凭证。不要上传、提交到 GitHub、写入截图或发给无关人员；网站让登录状态失效后仍需要重新登录，但不再需要手工寻找 Cookie。

## 从源码运行

### 运行环境

- 普通用户：使用 GitHub Release 中的 EXE，不需要安装 Python，也不需要创建 `.venv`。
- 源码运行或开发：Windows 10/11，Python 3.10 或更高版本，推荐 Python 3.12。
- 当前本机发布构建验证环境：Python 3.12.7、openpyxl 3.1.5、requests 2.32.3、websocket-client 1.8.0、PyInstaller 6.21.0。

项目使用根目录的 `.venv` 隔离运行和构建依赖。该目录已被 `.gitignore` 排除，不会上传到 GitHub。首次配置：

```powershell
cd "项目根目录"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r ".\02_源码和说明\requirements.txt"
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_spider_app.py"
```

也可以双击 `02_源码和说明\启动乙方宝爬虫工具.bat`，脚本会在缺少环境时自动创建 `.venv` 并安装依赖。

命令行运行爬虫：

```powershell
$env:YFB_COOKIE='在这里粘贴当前Cookie'
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_bid_spider.py" --days 7
```

指定 Excel 或仅演练：

```powershell
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_bid_spider.py" --days 31 --xlsx "D:\数据\招标信息.xlsx"
.\.venv\Scripts\python.exe ".\02_源码和说明\yfb_bid_spider.py" --days 31 --dry-run
```

每个关键词执行以下四种检索组合：

| 检索范围 | 检索方式 |
| --- | --- |
| 全文检索 | 智能检索 |
| 全文检索 | 精准检索 |
| 标题检索 | 智能检索 |
| 标题检索 | 精准检索 |

脚本在列表阶段只做必要的时间、地区、去重和关键词来源控制，尽量宽口径写入原始表；不再因为标题或摘要看起来不够相关就提前丢弃。最后由筛选后表统一执行严格业务规则，正文命中但标题不明确的项目会标记为“疑似相关”或“待终筛”，供人工复核。

## Excel 输出

1. 序号

3. 项目名称
4. 建设单位
5. 项目位置
6. 资质
7. 报名时间
8. 投标截止时间

资质和公告正文等长文本保存在“公告详情”工作表。主表中的“公告资质”或备注链接可跳转到对应详情，详情页也提供返回主表链接。筛选后文件包含“筛选删除记录”工作表，保留被终筛删除的项目、原始字段和筛出原因，便于人工追溯。

## 打包 Windows 软件

进入 `02_源码和说明`，双击 `打包成EXE.bat`。脚本会使用根目录的 `.venv`，安装 `requirements-dev.txt` 中固定版本的构建依赖，并在 `dist\乙方宝招标信息爬取工具` 生成成品。

手动打包命令：

```powershell
cd "项目根目录"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r ".\02_源码和说明\requirements-dev.txt"
cd ".\02_源码和说明"
..\.venv\Scripts\python.exe -m PyInstaller --noconfirm --clean --windowed --name "乙方宝招标信息爬取工具" --hidden-import yfb_bid_spider yfb_spider_app.py
```

正式发布时建议：

1. 使用不含真实业务数据的空白 Excel 模板。
2. 删除 Cookie、Token、备份表格、抓包文件和个人调试记录。
3. 将完整成品文件夹压缩为 ZIP，作为 GitHub Release 附件上传。
4. 源码仓库只保留必要源码、公开说明和脱敏模板。
5. 发布前在一台未配置 Python 的 Windows 电脑上试运行。

仓库中的 `.gitignore` 默认排除 `.venv`、本地成品、归档、构建缓存和 Excel 备份。发布 ZIP 时请在 GitHub Release 页面手动上传，不要为提交二进制文件而删除这些排除规则。

## 当前终筛规则

筛选后表会优先保留标题命中以下业务词的项目：水土保持、水保、绿色建筑评价、绿色建筑验收、绿建评价、绿建验收、绿色建筑、测绘、测量、多测合一、地形图、规划核实、房产实测、国土变更调查、不动产测绘、竣工测量、放线测量、验线测量、基坑监测、深基坑监测、第三方监测、专项监测、变形监测、沉降观测、沉降监测、结构监测、运营期结构监测、监测服务。

水土保持和绿色建筑类只要公告正文命中相关业务词，会直接保留。测绘/测量类要求正文同时命中测绘测量业务词和测绘乙级等资质表达。监测类要求正文同时命中监测业务词和工程测量、工程勘察、岩土工程、监测业绩等资质或业绩表达。

当前尚未把“物探、地下物探、工程物探、探测”作为正式保留规则；这类项目会进入原始表或删除记录，是否纳入需按业务边界另行确认。

## 已知限制

- 登录状态会过期，届时需在软件中重新登录；自动登录不可用时可改用手工 Cookie。
- 网站接口、字段或访问策略变化后，脚本可能需要更新。
- 部分公告正文可能无法完整获取，软件会保留链接并标记人工核对。
- 建设单位和时间字段来自不同公告格式的规则提取，无法保证全部自动识别。
- 为兼顾召回率，原始表会包含更多候选和无关结果；请优先查看筛选后表，并用“筛选删除记录”追溯边界项目。
- 软件版运行中的内部任务不能强制中断，需要等待本轮结束。

## 常见问题

**提示认证失败**

点击软件中的“登录乙方宝”重新登录；自动登录不可用时再粘贴新的手工 Cookie。

**Excel 保存失败**

先关闭正在由 Excel/WPS 打开的目标文件，再重新运行。

**没有新增数据**

可能是当前时间范围内没有符合条件的项目，也可能是项目已存在并被去重。可先使用演练模式查看日志。

**结果不完整或存在无关项目**

先检查“相关性”和“备注”列，再通过原公告链接人工确认。大范围采集后建议进行一次人工审核。

## 维护与贡献

主要配置位于 `02_源码和说明/yfb_bid_spider.py`：

- `SEARCH_KEYWORDS`：搜索关键词
- `AREA_IDS`：地区编号
- `TITLE_KEEP_TERMS`：终筛标题直接保留词
- `WATER_GREEN_TERMS`：水土保持/绿色建筑直接保留词
- `SURVEY_QUALIFICATION_PATTERN`：测绘测量资质匹配规则
- `MONITOR_QUALIFICATION_PATTERN`：监测类资质/业绩匹配规则
- `PROTECTED_TITLE_TERMS`：初筛噪声保护词
- `NOISE_TITLE_TERMS`：常规噪声词
- `STRICT_NOISE_TITLE_TERMS`：强噪声词

提交问题时请提供软件版本、时间范围、项目标题和脱敏后的日志。请勿提交 Cookie、Admin-Token 或包含个人业务数据的 Excel。

版本变化见 [CHANGELOG.md](CHANGELOG.md)，开发过程与维护记录见 [PROJECT_LOG.txt](PROJECT_LOG.txt)。
