# 项目文件清单

这个项目已经按用途整理好。平时只需要看 `01_直接使用软件` 和 `02_源码和说明`，旧资料在 `03_归档备份`。

## 01_直接使用软件

给日常使用和发布用。

- `乙方宝招标信息爬取工具/`：本机直接运行的软件目录，双击里面的 `乙方宝招标信息爬取工具.exe` 使用。
- 乙方宝招标信息爬取工具/2026-乙方宝招标信息统计.xlsx：本机日常使用的数据表，里面可能有你已经爬取过的数据。
- 乙方宝招标信息爬取工具/browser_data/：本机浏览器登录状态，不能发布或发给别人。
- `yifangbao-bid-spider-v1.2.0-windows-x64.zip`：最终 GitHub Release 发布包；已使用轻量环境构建并通过启动测试。

注意：不要只发单独的 exe，必须发 zip 或整个 `乙方宝招标信息爬取工具` 文件夹。

## 02_源码和说明

给开发、修改、源码运行、重新打包用。

- `yfb_bid_spider.py`：爬虫核心逻辑。
- yfb_spider_app.py：软件窗口界面。
- yfb_browser_auth.py：浏览器检测、自动登录和登录状态读取。
- `2026-乙方宝招标信息统计.xlsx`：空模板 Excel，用于打包和给新用户初始化。
- `requirements.txt`：运行依赖。
- `requirements-dev.txt`：开发和打包依赖。
- `启动乙方宝爬虫工具.bat`：用源码方式启动软件。
- `打包成EXE.bat`：重新打包 Windows 软件。
- `乙方宝爬虫使用说明.md` / `乙方宝爬虫使用说明.txt`：软件使用说明。

以后如果还要继续开发，主要就在这个目录里改。

## 03_归档备份

历史版本、旧包、开发资料和 Excel 备份放在这里；可再生成的构建调试文件已经清理。

这些文件一般不用打开，也不要发布到 GitHub。只有需要找回旧版本或对比历史数据时再看。

## 根目录文件

- `README.md`：GitHub 首页说明。
- `CHANGELOG.md`：版本更新记录。
- `PROJECT_LOG.txt`：项目开发记录。
- `项目文件清单.md`：项目文件说明。
- `目录说明.txt`：更偏日常使用的快速说明。
- `.gitignore`：控制哪些文件不上传到 GitHub。
- `.venv/`：本机 Python 虚拟环境，源码运行或打包时使用，不需要上传，也不需要复制给别人。
- `.git/`：Git 仓库数据，不要手动改。
- `.idea/`、`.agents/`、`.codex/`：本机工具配置，不用管，不要发布。

## 发布时上传什么

发布软件时，在 GitHub Release 附件里上传这个文件：

`01_直接使用软件/yifangbao-bid-spider-v1.2.0-windows-x64.zip`

不要上传 `.venv`、`.idea`、`.agents`、`.codex`、`03_归档备份`、`01_直接使用软件/乙方宝招标信息爬取工具` 整个文件夹。
